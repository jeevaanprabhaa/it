# AlgoTrader Web Terminal

## Overview
A full-stack Progressive Web App (PWA) trading terminal. Features live market data from Binance (with demo mode fallback), candlestick charts with indicators, real-time order book, paper trading with advanced order types, a trade journal with psychology analytics, a market heatmap, and full mobile responsiveness.

## Tech Stack
- **Frontend**: React 18 + TypeScript + Vite (port 5000) — PWA with service worker
- **Backend**: Node.js + Express + WebSocket (port 3001)
- **Charts**: lightweight-charts v4 (TradingView)
- **Analytics**: recharts v3 (journal psychology charts)
- **Market Data**: Binance public REST API + WebSocket streams (demo fallback if geo-blocked)

## Project Structure
```
client/
  public/
    manifest.json      - PWA manifest (AlgoTrader, theme #00ff88)
    sw.js              - Service worker (cache-first shell, network-first API)
    offline.html       - Offline fallback page
    icon-192.png       - PWA icon 192x192 (generated by generate-icons.js)
    icon-512.png       - PWA icon 512x512 (generated by generate-icons.js)
  src/
    components/        - Chart, OrderBook, SecuritiesTable, TradesPanel, OrderPanel,
                         JournalModal, JournalTab, HeatmapPage
    hooks/             - useMarketData (REST+WS), useIsMobile (breakpoint=768px)
    types/             - TypeScript type definitions
  generate-icons.js    - Node.js script to generate PNG icons (run once)
  index.html           - PWA meta tags, manifest link, apple-touch-icon
  vite.config.ts       - Vite config with proxy to backend
server/
  index.js             - REST API proxy + WebSocket relay + advanced order engine
run.sh                 - Startup script
```

## Features
- **PWA**: Installable on mobile and desktop; service worker caches app shell; offline fallback page; theme-color #00ff88
- **Candlestick Chart** with SMA 20, Bollinger Bands, and VWAP overlays (toggle each independently)
- **Order Book / Market Depth** with real-time bid/ask levels
- **Advanced Order Types**: Market (immediate), Limit (fills when price reaches trigger), Stop-Loss (sells below stop) — backend monitors pending orders every 30s
- **Browser Notifications**: When a limit/stop order fills, browser notification fires and journal modal pops up
- **Trade Journal**: Log reason, emotion, market condition, notes per trade; Psychology sub-page with charts
- **Market Heatmap**: 20 crypto assets colored by % change, sized by volume; auto-refreshes every 30s
- **Mobile Responsive** (< 768px):
  - Bottom tab bar: Charts 📈 | Scanner 🔍 | Portfolio 💼 | Journal 📓
  - Charts tab: full-screen chart + floating ⇅ FAB to open buy/sell panel
  - Scanner: securities table (tapping a symbol navigates back to chart)
  - Heatmap: 2-column card grid instead of treemap
- **Demo Mode**: Auto-activates with realistic synthetic data if Binance is geo-restricted

## Running the Project
```bash
bash run.sh
```
Starts: Express backend (port 3001) + Vite dev server (port 5000)

## Notes
- Binance API returns 451 (geo-restricted) from Replit — demo mode activates automatically.
- Service worker only caches in production builds; in dev it registers but Vite's HMR takes priority.
- Icons are pre-generated PNGs in client/public/. Re-run `node client/generate-icons.js` to regenerate.
- Advanced orders use in-memory storage (not persisted across server restarts).
- WebSocket path /ws is shared by frontend clients and Binance stream relay on the backend.
